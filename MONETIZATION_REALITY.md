## 2. 웹 vs 앱 광고 전략 (정책 기준)
| 표면 | 광고 | 비고 |
|---|---|---|
| 블로그(콘텐츠) | **AdSense** 수동 유닛 | 진짜 수익원. 디스플레이 |
| 앱(retwork.jp PWA) | **자체 광고(local_ads)** + (추후 AdMob) | AdSense 앱 안 게재는 **정책 위반** |
| 랜딩 | (광고 자제) | 게이트웨이라 광고 부적합 |

- AdSense=웹 전용(JS 태그), AdMob=앱 전용(네이티브 SDK). **같은 화면 동시 사용 X, 환경별 분기.**
- 한 구글 계정에 AdSense+AdMob 둘 다 연결·정산 통합 가능.

## 3. 리워드 광고("보고 치리 받기") — 가능/불가능의 진실
- **리워드 모델 자체는 합법.** AdSense에도 공식 "리워드 광고 정책" 존재.
- **단, 리워드 "동영상"은 AdMob(앱) 기능. AdSense(웹/PWA)엔 그 형식이 없음.**
- 핵심 원리: **"광고주가 리워드인 줄 알고 산 인벤토리"에서만 보상 가능.** 일반 광고에 보상 주면 무효 트래픽(사기) → **계정 정지**.
  - 자체 광고(local_ads) = 본인이 광고주 → 보상 줘도 **합법** (지금 PWA가 이거).
  - AdSense 일반 광고에 보상 = ❌
  - AdMob 리워드 동영상 = ✅ (단 네이티브 앱)
- **PWA는 AdMob 직접 불가** → **Capacitor로 네이티브 래핑** 필요. (TWA는 광고 SDK 못 붙임 → AdMob 목적이면 TWA 아니라 **Capacitor**)

### 플랫폼별 경로
```
웹/PWA(모든 브라우저·iOS 포함) → 자체광고 리워드 + AdSense(블로그)   ← 항상, 지금
Android 앱  → Capacitor 래핑 + AdMob 리워드 (Play $25 1회)
iOS         → 1단계 PWA(자체광고) → 2단계 Capacitor+App Store+AdMob ($99/년, ATT, 심사 빡빡)
```

## 4. ⚠️ 치리 보상 설계 리스크 (별개로 중요)
- 치리 → **商品券(상품권) 교환**은 (a) 리워드 광고 정책 "비현금·앱내전용·양도불가" 위반 소지, (b) **일본 자금결제법**(환금성 포인트) 규제 소지.
- → 치리는 **앱 내 전용 보상**(프리미엄·기능 등)으로 두는 게 안전. (AdMob 쓰든 안 쓰든 법률 영역, 전문가 확인 권장)

## 5. 광고 단가 감 (업계 추정, 일본≈준-Tier1)
| 형식 | eCPM(Tier-1) |
|---|---|
| 배너 | $0.5~2 |
| AdSense 디스플레이(웹) | $1~8 RPM |
| 전면 | $6~12 |
| **🏆 리워드 동영상(AdMob)** | **$15~45** |
- 리워드가 단가 최강(배너 10~30배). 단 단위가 다름(완료 시청당 vs 페이지뷰당). 신규 서비스는 현실 수익을 상한의 20~50%로 보수적으로.

## 6. 코드: AdMob 전환 "교체 지점" (나중에 작업용)
- 현재 광고 모달이 **이미 리워드 UX 하네스**임: `openWatchAdPage`(opt-in) → `ctOpenFullscreenAd`(전체화면 + `_injectAd` 자체광고 + **15초 카운트다운**) → 완료 시 +치리.
- **AdMob 전환 = `ctOpenFullscreenAd` 안의 `_injectAd`+15초 타이머 한 군데만** AdMob 리워드 SDK 호출로 교체:
  - load → show → `onUserEarnedReward` 콜백 → +치리 / 중도종료→보상X / no-fill→폴백.
  - 환경 분기: `Capacitor.isNativePlatform()` → 앱이면 AdMob, 브라우저면 현행 자체광고.
- `AD_CONFIG.enabled=false` 유지(앱 AdSense 켜지 말 것). `reward`/`modal` 슬롯에 AdSense 주입 금지.
